# STM32 智能小车 — 红外循迹 + 超声波避障

> 基于 STM32F103C8T6 的四路红外循迹 + 超声波避障智能小车，支持蓝牙遥控、OLED 显示、舵机云台。

## 硬件清单

| 模块 | 型号 | 数量 |
|------|------|------|
| 主控 | STM32F103C8T6 (Cortex-M3) | 1 |
| 电机驱动 | TB6612FNG | 1 |
| 直流电机 | TT 马达 1:48 | 4 |
| 红外传感器 | TCRT5000 × 4 路 | 1 |
| 超声波 | HC-SR04 | 1 |
| 舵机 | SG90 (云台) | 1 |
| OLED | 0.96寸 I2C (SSD1306) | 1 |
| 蓝牙 | JDY-31 (BLE) | 1 |

## 模式与蓝牙指令

| 指令 | 模式 | 说明 |
|------|------|------|
| `1` | 自动模式 | 无障碍 → 循迹，有障碍 → 避障（默认） |
| `2` | 纯循迹 | 四路红外跟踪黑线 |
| `3` | 纯避障 | 超声波测距 + 舵机扫描绕障 |
| `4` | 蓝牙遥控 | F=前 B=后 L=左 R=右 S=停 |

`1~4` 在任何模式下都生效。`F/B/L/R/S` 仅在蓝牙模式(4)下生效。

## 循迹算法

4 路红外传感器分级响应，9 种情况按优先级匹配：

```
[1 0 0 1] 完美居中 → 直走
[1 0 1 1] X2踩线  → 左微调 (差速)
[1 1 0 1] X3踩线  → 右微调 (差速)
[0 0 1 1] 左偏    → 急左转 (坦克式)
[1 1 0 0] 右偏    → 急右转 (坦克式)
[0 1 1 1] 极左偏  → 半速左转修正
[1 1 1 0] 极右偏  → 半速右转修正
3路以上踩线    → 十字路口，直走
[1 1 1 1] 离线    → 原地找线（超时2秒直走）
```

## 避障算法

1. 超声波测距 → 前方 < 25cm 触发
2. 舵机左扫(0°) → 右扫(180°) → 比较两侧距离
3. 选择空旷方向 (> 35cm) → 旋转 → 前进绕行
4. 两侧均堵 → 180° 掉头

## 面试常见问题

### 电机 & PWM

**Q: 为什么电机 PWM 频率选 5KHz？**

低于 1~2KHz 人耳会听到电机啸叫，高于 20KHz 电机驱动芯片 TB6612 开关损耗增大、发热明显。5KHz 在两者之间：人耳基本听不到，驱动芯片也扛得住。公式：`72MHz ÷ ((psc+1) × (arr+1)) = 72MHz ÷ (2 × 7200) = 5KHz`。

**Q: 舵机信号为什么必须是 50Hz？**

SG90 舵机用 0.5ms~2.5ms 高电平对应角度 0°~180°，这个高电平的参考周期是 20ms（=50Hz）。如果频率不对，舵机内部比较器无法正确解码脉宽，会抖动或卡死。代码里 `Servo_Init(1999, 719)`：72MHz ÷ 720 = 100KHz 计数器时钟，100KHz ÷ 2000 = 50Hz。

### 循迹算法

**Q: 循迹是怎么设计的？为什么不用 PID？**

4 路红外输出 0/1 离散值，不是连续偏移量，没法直接套 PID。所以用了 9 种情况的分级匹配策略：常见情况放前面早命中，极端情况兜底。虽然分辨率只有 4 级，但够用——代价是急弯容易丢线。换摄像头做连续反馈的话就会上 PID。

**Q: 差速转向怎么实现？和普通舵机转向有什么区别？**

像坦克——左右轮速度差产生转向。微调时一边慢一边快（差速），急转时一个正转一个反转（坦克式，转弯半径为零）。和舵机转向的区别：不需要额外转向机构，靠两个电机就能实现前进、后退、原地旋转。

**Q: 全部离线时原地打转那段代码有什么隐患？怎么解决的？**

原代码 `while(X2==1 && X3==1)` 没超时保护，找不到黑线就永远出不来。我加了超时计数器 `timeout < 2000`（约 2 秒），超时就直走，防止车在没有跑道的场地上死循环。

### 避障算法

**Q: 超声波怎么测距？**

Trig 发 10μs 高电平脉冲 → HC-SR04 自动发 8 个 40KHz 超声波 → Echo 拉高 → TIM4 开始计时 → 收到回波后 Echo 拉低 → 停止计时。距离 = 高电平时间(μs) ÷ 58。4 米内有效，超出返回 0。

**Q: 同时遇到黑线和障碍物，优先哪个？**

避障。`Control_AutoMode()` 里先判断 `obstacleDist < 25cm`，有障碍直接进避障流程，没障碍才走循迹。逻辑很简单：前面有墙，不管黑线在哪都得先躲。

### 中断 & 通信

**Q: `volatile` 是干什么的？你项目里哪用到了？**

告诉编译器"每次用这个变量必须从内存重新读，不能缓存到寄存器里"。项目里 `RxData` 加了 `volatile`——它在 USART1 中断里被写入，在主循环里被读取。不加的话编译器优化后主循环可能永远看不到中断更新的新值，蓝牙指令就收不到了。

**Q: 串口接收中断为什么只处理了 RXNE？**

RXNE（接收非空）是关键中断，但生产级代码还应该处理 ORE（溢出错误），否则线路干扰导致溢出后，RXNE 可能再也触发不了，蓝牙永久失联。本项目简化处理，重点演示中断和主循环的协作机制。

### STM32 基础

**Q: STM32F103C8T6 的系统时钟是多少？TIM 时钟怎么算？**

外接 8MHz 晶振 → PLL 倍频到 72MHz（`SystemInit()` 默认）。TIM1/2/3/4 挂 APB1（最大 36MHz），但因为 APB1 预分频器 ≠ 1，定时器时钟自动 ×2 = 72MHz。所以定时器的时基还是 72MHz。

**Q: I2C 怎么驱动 OLED 的？为什么用软件 I2C 而不是硬件？**

用 GPIO 模拟 I2C 时序（软件 I2C）：拉低 SDA → 拉低 SCL → 发 8 位数据 → 拉高 SCL 锁存。不用硬件 I2C 的原因：F103 的硬件 I2C 有历史 bug（在特定时序下会卡死），软件 I2C 更稳定且移植性好。

### 系统设计

**Q: 如果让你重做这个项目，你会怎么改进？**

1. 加 K210/OpenMV 摄像头做视觉循迹，输出连续偏移量 → 上 PID
2. 6 路红外替代 4 路，提高离散分辨率
3. 加 MPU6050 陀螺仪做航向角反馈，转弯不再靠延时估算
4. FreeRTOS 多任务替代裸机 while 循环
5. 串口中断加 ORE/溢出处理，通信更健壮
